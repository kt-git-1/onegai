# おねがいチャリン Priority 1 UI状態仕様 v1.0

## 1. 目的

Priority 1画面の通常、処理中、エラー、空状態、権限差分を実装可能な粒度で定義する。

- 情報設計と遷移：`futari_charin_priority1_wireframes.md`
- 色、書体、余白、角丸：`futari_charin_priority1_visual_foundation.md`

## 2. 共通状態

### 2.1 ボタン

| 状態 | 表示 | 操作 |
|---|---|---|
| 通常 | `primary` 背景、`textPrimary` 文字 | タップ可能 |
| 押下中 | `primaryPressed` 背景 | タップ継続中のみ表示 |
| disabled | `primarySoft` 背景、`textDisabled` 文字 | タップ不可 |
| 処理中 | 文言を維持し、右側にProgressView | 二重送信不可 |

### 2.2 入力欄

| 状態 | 表示 |
|---|---|
| 通常 | `border` 1 pt |
| フォーカス | `primary` 2 pt |
| 入力エラー | `error` 2 pt、直下にエラー文言 |
| disabled | `primarySoft` 面、`textDisabled` 文字 |

- 入力エラーは最初の送信時またはフォーカス離脱時に表示する。
- 入力中に条件を満たした項目はエラー表示を解除する。

### 2.3 通信

- 初回読込：主要レイアウトを保つスケルトンを表示する。
- 更新中：現在値を残し、対象領域だけProgressViewを表示する。
- 画面全体の通信エラー：本文と `もう一度試す` を表示する。
- 送信エラー：入力値や選択状態を維持し、再送可能にする。
- 同じ操作の二重送信を禁止する。

### 2.4 フィードバック

- 軽い成功：画面下部のトースト。
- 次の行動が必要な成功：成功画面または自動遷移。
- 入力に紐づくエラー：該当入力欄の直下。
- 画面全体のエラー：コンテンツ領域内。
- 破壊的操作や期限切れ：赤色だけに依存せず文言を併記する。

## 3. 初回導線

### 3.1 オンボーディング 1〜3

- 通常：現在ページとCTAを表示。
- 押下中：CTAを押下状態にする。
- 画面遷移中：CTAの再タップを無効化する。
- 3画面でキャラのサイズ、水平位置、イラスト枠を統一する。

### 3.2 登録 / ログイン

- 通常：3つの認証方法を表示。
- 認証開始中：選択した認証ボタンだけ処理中にし、ほかをdisabledにする。
- 認証キャンセル：通常へ戻し、エラーは表示しない。
- 認証失敗：ボタン群の下にエラーと再試行可能な状態を表示。
- 既存ユーザー：プロフィール、招待、参加状態に応じて既定画面へ振り分ける。

### 3.3 メール登録

- 未入力：CTA disabled。
- 入力中：各値を保持し、形式エラーを入力欄ごとに表示。
- 有効：メール形式、パスワード条件、一致条件を満たすとCTA enabled。
- 登録中：CTAを処理中にし、全入力欄を編集不可にする。
- 登録失敗：入力値を保持し、原因に対応するエラーを表示。
- 成功：プロフィール設定へ自動遷移。

### 3.4 プロフィール設定

- 初期：絵文字は未選択、名前は空、CTA disabled。
- 名前入力済み：絵文字の選択有無にかかわらずCTA enabled。
- 名前が空白文字のみ：CTA disabled。
- 保存中：CTAを処理中にし、入力とアイコン選択を無効化。
- 保存失敗：入力内容を保持し、再送可能にする。
- 成功：テンプレート適用へ自動遷移。

### 3.5 テンプレート適用

- 通常：テンプレート内容とCTAを表示。
- 作成中：CTAを処理中にし、二重作成を禁止。
- 失敗：既定エラー文言と `もう一度試す` を表示。
- 成功：招待画面へ自動遷移。

### 3.6 招待画面

- 通常：招待コード、LINE共有、リンクコピーを表示。
- 共有シート表示中：元画面の状態を保持。
- コピー成功：`招待リンクをコピーしました` トースト後、招待待ちへ遷移。
- LINE共有完了：招待待ちへ遷移。
- 共有キャンセル：招待画面に留まる。
- 共有失敗：コードを保持し、再試行可能にする。

### 3.7 招待待ち

- 待機中：待機表示、招待コード、共有操作を表示。
- 参加検知中：画面を維持し、自動更新する。
- オフライン：待機表示を維持し、接続エラーのバナーを表示。
- 招待期限切れ：同じ画面で新しい招待を再発行できるCTAを表示。
- 再発行中：CTAを処理中にし、二重発行を禁止。
- 再発行成功：新しいコードとリンクへ表示を更新し、待機状態を継続。
- 参加完了：ホームへ自動遷移。
- ログアウト中：操作を無効化し、完了後に登録 / ログインへ戻る。
- 待機中はgroupのmemberIdsをリアルタイム監視し、2人になった時点で最新セッションを取得してホームへ遷移する。

### 3.8 招待参加

- 招待確認：認証前でも招待元と招待内容、`参加する` を表示する。
- 未ログイン：`参加する` から登録 / ログインへ進み、対象の招待情報を保持する。
- プロフィール未設定：認証後にプロフィール設定を必須表示し、対象の招待情報を保持する。
- 参加可能：認証とプロフィール設定の完了後にgroup参加処理を開始する。
- 参加処理中：CTAを処理中にし、二重参加を禁止。
- 期限切れ：`この招待は期限切れです` を表示。
- 2人参加済み：`この招待は使用できません` を表示。
- 参加失敗：エラーと `もう一度試す` を表示。
- 参加成功：ホームへ自動遷移。
- 招待された側では、テンプレート適用と新しい招待の作成を表示しない。

## 4. 本体画面

### 4.1 ホーム：自分 / ふたり

- 初回読込：ヘッダーとタブを残し、カード領域をスケルトン表示。
- 通常：残高、最も近い未交換券、進捗、お願い、最近のきろくを表示。
- 交換可能：対象券と `交換する` を貯金箱カード内に表示。
- 残高不足：残りコインを表示し、`交換する` は表示しない。
- ごほうび券なし：作成用の空状態をカード内に表示。
  - CTA：`ごほうび券を作る`
  - CTAタップ：ごほうび券作成へ遷移。
- きろくなし：最近のきろく領域に空状態を表示。
- 更新失敗：直前の表示を保持し、再読込可能なバナーを表示。
- 自分 / ふたり切り替え中：横位置とカード高を固定し、レイアウトを揺らさない。

### 4.2 お願い一覧

- 初回読込：カードリストをスケルトン表示。
- 通常：お願いをよく使う順で表示。
- 空状態：`まだお願いがありません` と `お願いを作る` を表示。
- 作成者がカードをタップ：カード内に編集操作を表示。
- 作成者以外がカードをタップ：編集操作を表示しない。
- ちゃりん開始：対象カードだけ処理中にし、確認ダイアログの重複表示を防ぐ。
- 読込失敗：エラーと `もう一度試す` を表示。

### 4.3 ちゃりん確認ダイアログ

- 通常：対象、コイン、入金先、目標を表示。
- 送信中：確定ボタンを処理中、キャンセルをdisabledにする。
- 送信失敗：ダイアログを閉じず、エラーを表示して再送可能にする。
- 成功：ちゃりん演出へ遷移。

### 4.4 ちゃりん演出

- 通常成功：1〜2秒の演出、増加量、残高、次の目標を表示。
- 交換可能到達：交換可能文言と `交換する` を表示。
- 取り消し可能：10秒間、画面下部に `取り消す` を表示。
- 取り消し中：トースト操作をdisabledにし、ProgressViewを表示。
- 取り消し成功：残高を戻し、ホームへ遷移。
- 取り消し失敗：現在残高を再取得し、エラーを表示。
- 時間経過：ホームへ自動遷移し、取り消しトーストはホーム上で残り時間だけ継続表示する。

### 4.5 ごほうび券一覧

- 初回読込：フィルターを残し、カードをスケルトン表示。
- 通常：選択中の分類と貯金箱に一致する券を表示。
- 残高不足：残りコインを表示し、`交換する` は非表示。
- 交換可能：交換可能ラベルと `交換する` を表示。
- フィルター結果なし：条件に対応した空状態を表示。
- 読込失敗：エラーと `もう一度試す` を表示。

### 4.6 持っている券

- 初回読込：カードをスケルトン表示。
- 使える券：通常カードと `券を表示` を表示。
- 使った券：カードをグレーアウトし、使用日を表示。
- 空状態：`まだ持っている券がありません` を表示。
- 読込失敗：エラーと `もう一度試す` を表示。

### 4.7 きろく

- 初回読込：月次サマリーとタイムラインをスケルトン表示。
- 通常：日付ごとのタイムラインを表示。
- フィルター適用中：選択値を維持し、タイムラインだけ更新する。
- 空状態：`まだきろくがありません` を表示。
- 相手のちゃりん：スタンプCTAを表示。
- 自分のちゃりん：スタンプCTAを表示しない。
- スタンプ選択中：対象行に紐づく小さなポップオーバーを表示。
- スタンプ送信中：選択したスタンプを一時表示し、追加操作を無効化。
- スタンプ失敗：直前の表示へ戻し、エラートーストを表示。

## 5. 確定履歴

- 招待待ち中に招待が期限切れになった場合、同じ画面で新しい招待を再発行する。
- ちゃりん演出からホームへ戻った後も、10秒取り消しを残り時間だけ継続する。
- ホームでは未交換券から最も近い1件を自動表示し、券がない場合は `ごほうび券を作る` CTAを表示する。
- 同じ券は1回だけ交換でき、交換後は次の未交換券へ自動更新する。
