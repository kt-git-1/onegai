# ふたりちゃりん（仮）Priority 1 ワイヤー仕様 v1.0

## 1. 目的

Priority 1のFigma画面を作るためのワイヤー仕様。

このドキュメントでは、画面ごとの情報設計、主要CTA、状態、遷移を定義する。
新しい仕様判断が必要な箇所は「要確認」として記載し、未確認のまま決定扱いにしない。

---

## 2. 前提

### 2.1 対象画面

1. オンボーディング1
2. オンボーディング2
3. オンボーディング3
4. 登録 / ログイン
5. プロフィール設定
6. テンプレート適用
7. 招待画面
8. 招待待ち画面
9. 招待リンク受信側の参加画面
10. ホーム：自分の貯金箱
11. ホーム：ふたりの貯金箱
12. お願い一覧
13. ちゃりん確認ダイアログ
14. ちゃりん演出画面
15. ごほうび券一覧
16. 持っている券
17. きろく通常

### 2.2 合意済み方針

- 招待は初回導線に必須で入れる
- 招待完了前はホームに入れない
- 個人だけで使える状態は作らない
- ちゃりんは「自分が相手にしてあげたこと」または「ふたりで一緒にやったこと」を自分で記録する
- 個人貯金箱とふたりの貯金箱を持つ
- ホームの初期表示は自分の貯金箱
- ごほうび券は交換前のマスター、持っている券は交換後のチケットとして扱う

### 2.3 共通レイアウト

画面サイズ想定：
- iPhone 15 / 393 x 852 相当

共通トーン：
- クリーム背景
- 白カード
- コイン系の黄色 / オレンジ
- ハート系のアクセント
- 絵文字アイコン
- 子どもっぽすぎない、カップルで使って恥ずかしくないかわいさ

共通ナビゲーション：
- 初回導線中は下部タブなし
- 招待完了後の本体画面は下部タブあり
- 下部タブ：ホーム / お願い / ごほうび / きろく
- 設定導線はホーム右上の歯車アイコン

## 4. 画面別ワイヤー

## 4.1 オンボーディング1：お願いをちゃりんする

### 目的

「お願いを叶えたらちゃりんする」という基本行動を伝える。

### 配置

上から：
1. ステータスバー余白
2. ページインジケーター `1 / 3`
3. メインビジュアル
   - 貯金箱、コイン、お願いカードを想起できるイラスト
4. 見出し
   - `お願いを叶えたら、`
   - `ちゃりんしよう。`
5. 本文
   - `マッサージ、皿洗い、買い出し。`
   - `ふたりの毎日の“してくれて嬉しい”を`
   - `コインにできます。`
6. 下部固定CTA
   - `次へ`

### 遷移

- `次へ` → オンボーディング2

## 4.4 登録 / ログイン

### 目的

ユーザー登録またはログインを行う。

### 配置

上から：
1. アプリロゴまたは貯金箱アイコン
2. アプリ名
   - `ふたりちゃりん`
3. コピー
   - `やさしさが、ちゃりんと貯まる。`
4. 認証ボタン群
   - `Appleで続ける`
   - `Googleで続ける`
   - `メールで登録`

   - `すでにアカウントをお持ちの方はログイン`

   - 利用規約

### 遷移

- 新規登録完了 → プロフィール設定

- 既存ログインで招待未完了 → 招待待ち画面または招待画面

### 要確認

- メール登録画面をPriority 1で別画面として作るか、登録 / ログイン画面内のボタンだけに留めるか。

## 4.6 テンプレート適用

### 目的

初期のお願い、ごほうび券、貯金箱を作成する。

### 配置

上から：
1. 見出し
   - `まずはテンプレートで始めよう`
2. 本文
   - `カップル向けテンプレートには、`
   - `お願い・ふたりのお願い・ごほうび券が`
   - `最初から入っています。`
3. テンプレート内容プレビュー
   - `お願い`
     - `💆 マッサージ10分`
     - `🧽 皿洗い`
   - `ふたりのお願い`
     - `🧹 ふたりで部屋を片付ける`
   - `ごほうび券`
     - `🍰 コンビニスイーツごほうび券`
     - `☕ スタバごほうび券`
4. 下部固定CTA
   - `カップル向けテンプレートを使う`

### 状態

- 通常
- 作成中
  - CTAにローディング
  - 二重送信不可
- エラー
  - `テンプレートを作成できませんでした。時間をおいてもう一度お試しください。`

### 遷移

- 作成成功 → 招待画面

1. 見出し

## 4.8 招待待ち画面

### 目的

相手の参加完了を待つ。招待完了前にホームへ入れないことを自然に伝える。

### 配置

上から：
1. 状態アイコン
   - ふたりの貯金箱またはハート付き貯金箱
2. 見出し
   - `相手の参加を待っています`
3. 本文
   - `相手が参加すると、`
   - `ふたりの貯金箱を始められます。`
4. 招待コードカード
   - `ABCD-1234`
   - コピーアイコンボタン
5. CTA群
   - `LINEで再送する`
   - `招待リンクをコピー`
6. 下部テキストボタン
   - `ログアウト`

### 状態

- 待機中
- コピー完了
- 参加検知中
- 参加完了

### 遷移

- 相手の参加完了を検知 → ホーム
- `LINEで再送する` → OS共有 / LINE共有
- `招待リンクをコピー` → コピー完了トースト
- `ログアウト` → ログアウト確認

### 要確認

- 参加完了時に自動でホームへ遷移するか、`はじめる` ボタンを表示して遷移するか。

## 4.14 ちゃりん演出画面

### 目的

ちゃりん完了の気持ちよさと、残高更新を見せる。

### 表示形式

- 全画面
- 1〜2秒程度
- 画面下部に30秒取り消しトーストを出す

### 配置

通常時：
1. コイン投入アニメーション領域
2. 大きい文言
   - `ちゃりん！`
3. コイン増加
   - `+100コイン`
4. 残高
   - `520 → 620`
5. 目標情報
   - `あと80コインで`
   - `スタバごほうび券`
6. 取り消しトースト
   - `ちゃりんしました`
   - `取り消す`

交換可能になった場合：
1. `ちゃりん！`
2. `+100コイン`
3. `スタバごほうび券が`
4. `交換できるようになりました`
5. CTA
   - `交換する`
6. 取り消しトースト
   - `取り消す`

### 遷移

- 一定時間後 → 元画面またはホーム
- `取り消す` → cancelCharin
- `交換する` → 交換確認またはごほうび券一覧

### 要確認

- 演出後の戻り先を「元画面」にするか「ホーム」にするか。

## 6. 要確認一覧

1. メール登録画面をPriority 1で別画面として作るか、登録 / ログイン画面内のボタンだけに留めるか。
2. プロフィールの絵文字アイコンは初期値をランダムに入れるか、未選択状態にするか。
3. 招待共有後に自動で招待待ち画面へ遷移するか、`招待待ちへ` ボタンを押させるか。
4. 招待参加完了時に自動でホームへ遷移するか、`はじめる` ボタンを表示して遷移するか。
5. 招待された側にもプロフィール未設定フローを必須にするか。
6. ホームの `交換する` はごほうび券一覧に遷移するか、対象券の交換確認に直接遷移するか。
7. お願いカード全体タップで編集に入るか、作成者のみ編集ボタンを表示するか。
8. ちゃりん演出後の戻り先を「元画面」にするか「ホーム」にするか。
9. ごほうび券カードの残高不足時は `交換する` disabledにするか、ボタン文言を `あと180コイン` にするか。
10. Priority 1でスタンプ選択UIを同一画面内の小さなポップオーバーとして表現するか、Priority 2画面として分けるか。
