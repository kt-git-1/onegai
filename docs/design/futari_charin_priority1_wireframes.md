# おねがいチャリンPriority 1 ワイヤー仕様 v1.0

## 1. 目的

Priority 1のFigma画面を作るためのワイヤー仕様。

このドキュメントでは、画面ごとの情報設計、主要CTA、状態、遷移を定義する。
仕様判断が確定した項目は、末尾の確定済み仕様メモにまとめる。

---

## 2. 前提

### 2.1 対象画面

1. オンボーディング1
2. オンボーディング2
3. オンボーディング3
4. 登録 / ログイン
5. メール登録
6. プロフィール設定
7. テンプレート適用
8. 招待画面
9. 招待待ち画面
10. 招待参加画面
11. ホーム：自分の貯金箱
12. ホーム：ふたりの貯金箱
13. お願い一覧
14. ちゃりん確認ダイアログ
15. ちゃりん演出画面
16. ごほうび券一覧
17. 持っている券
18. きろく通常

### 2.2 合意済み方針

- 招待は初回導線に必須で入れる
- 招待完了前はホームに入れない
- 個人だけで使える状態は作らない
- ちゃりんは「自分が相手にしてあげたこと」または「ふたりで一緒にやったこと」を自分で記録する
- 個人貯金箱とふたりの貯金箱を持つ
- ホームの初期表示は自分の貯金箱
- ごほうび券は交換前のマスター、持っている券は交換後のチケットとして扱う

### 2.3 共通レイアウト

画面サイズ想定：
- iPhone 15 / 393 x 852 相当

共通トーン：
- クリーム背景
- 白カード
- コイン系の黄色 / オレンジ
- ハート系のアクセント
- 絵文字アイコン
- 子どもっぽすぎない、カップルで使って恥ずかしくないかわいさ

共通ナビゲーション：
- 初回導線中は下部タブなし
- 招待完了後の本体画面は下部タブあり
- 下部タブ：ホーム / お願い / ごほうび / きろく
- 設定導線はホーム右上の歯車アイコン

## 4.1 オンボーディング1：お願いをちゃりんする

### 目的

「お願いを叶えたらちゃりんする」という基本行動を伝える。

### 配置

上から：
1. ステータスバー余白
2. ページインジケーター `1 / 3`
3. メインビジュアル
   - 貯金箱、コイン、お願いカードを想起できるイラスト
4. 見出し
   - `お願いを叶えたら、`
   - `ちゃりんしよう。`
5. 本文
   - `マッサージ、皿洗い、買い出し。`
   - `ふたりの毎日の“してくれて嬉しい”を`
   - `コインにできます。`
6. 下部固定CTA
   - `次へ`

### 遷移

- `次へ` → オンボーディング2

### 配置

2. 見出し

3. 入力フォーム

   - `登録すると、利用規約とプライバシーポリシーに同意したことになります。`

## 4.6 プロフィール設定

### 目的

アプリ内で表示する名前と絵文字アイコンを設定する。

### 配置

上から：
1. ナビゲーション領域
2. 見出し
   - `プロフィールを設定しよう`
3. 補足
   - `相手に表示される名前とアイコンです。`
4. アイコン選択
   - 初期状態は未選択
   - 選択は任意
   - 未選択時はプレースホルダー表示
   - 選択後は大きい絵文字プレビュー
   - `アイコンを選ぶ`
5. 名前入力
   - ラベル：`名前`
   - プレースホルダー：`例：花男`
6. 下部固定CTA
   - `保存して次へ`

### 状態

- 名前未入力時：CTA disabled
- 名前が空白文字のみ：CTA disabled
- 名前入力済み：絵文字の選択有無にかかわらずCTA enabled

### 遷移

- `保存して次へ` → テンプレート適用

## 4.9 招待待ち画面

### 目的

相手の参加完了を待つ。招待完了前にホームへ入れないことを自然に伝える。

### 配置

上から：
1. 状態アイコン
   - ふたりの貯金箱またはハート付き貯金箱
2. 見出し
   - `相手の参加を待っています`
3. 本文
   - `相手が参加すると、`
   - `ふたりの貯金箱を始められます。`
4. 招待コードカード
   - `ABCD-1234`
   - コピーアイコンボタン
5. CTA群
   - `LINEで再送する`
   - `招待リンクをコピー`
6. 下部テキストボタン
   - `ログアウト`

### 状態

- 待機中
- コピー完了
- 参加検知中
- 招待期限切れ
  - `招待の期限が切れました`
  - CTA：`新しい招待を発行する`
- 招待再発行中
  - CTAにローディング
  - 二重発行不可
- 参加完了

### 遷移

- 相手の参加完了を検知 → ホームへ自動遷移
- `LINEで再送する` → OS共有 / LINE共有
- `招待リンクをコピー` → コピー完了トースト
- `新しい招待を発行する` → 同じ画面でコードとリンクを更新して待機を継続
- `ログアウト` → ログアウト確認

## 4.10 招待参加画面

### 目的

招待リンクまたは招待コードで特定した招待内容を確認し、認証後にグループへ参加する。

### 配置

上から：
1. 招待されたことが分かるアイコン
2. 見出し
   - `おねがいチャリンに招待されました`
3. 招待元情報
   - `花男さんからの招待`
4. 説明
   - `参加すると、ふたりの貯金箱とごほうび券を一緒に使えます。`
5. 参加CTA
   - `参加する`
6. 補足
   - `参加にはログインが必要です。`

### 状態

- 未ログイン
  - `参加する` → 登録 / ログインへ
- 認証中 / プロフィール設定中
  - 招待IDを保留状態として保持する
- ログイン済み
  - プロフィール設定済みならgroup参加処理へ進む
  - プロフィール未設定ならプロフィール設定へ進む
- 招待期限切れ
  - `この招待は期限切れです`
- すでに2人参加済み
  - `この招待は使用できません`

### 遷移

- 招待リンク受信または招待コード照合成功 → 招待参加画面
- `参加する`、かつ未ログイン → 登録 / ログイン
- 認証成功、かつプロフィール未設定 → プロフィール設定
- 認証成功、かつプロフィール設定済み → group参加処理
- プロフィール設定完了 → 保持している招待でgroup参加処理
- group参加成功 → ホーム

### 処理上の制約

- 認証前にはgroup参加を確定しない。
- 登録 / ログインやプロフィール設定へ遷移しても、対象の招待情報を失わない。
- 招待された側はテンプレート適用、招待画面、招待待ち画面を経由しない。

## 4.15 ちゃりん演出画面

### 目的

ちゃりん完了の気持ちよさと、残高更新を見せる。

### 表示形式

- 全画面
- 1〜2秒程度
- 画面下部に10秒取り消しトーストを出す

### 配置

通常時：
1. コイン投入アニメーション領域
2. 大きい文言
   - `ちゃりん！`
3. コイン増加
   - `+100コイン`
4. 残高
   - `520 → 620`
5. 目標情報
   - `あと80コインで`
   - `スタバごほうび券`
6. 取り消しトースト
   - `ちゃりんしました`
   - `取り消す`

交換可能になった場合：
1. `ちゃりん！`
2. `+100コイン`
3. `スタバごほうび券が`
4. `交換できるようになりました`
5. CTA
   - `交換する`
6. 取り消しトースト
   - `取り消す`

### 遷移

- 一定時間後 → ホーム
- ホームへ戻った後も、10秒の残り時間だけ取り消しトーストを継続表示
- `取り消す` → cancelCharin
- `交換する` → 対象券の交換確認

## 6. 確定済み仕様メモ

1. メール登録画面はPriority 1で別画面として作る。
2. プロフィールの絵文字アイコンは初期値を未選択状態にする。
   - 絵文字アイコンは任意入力とする。
3. 招待共有後は自動で招待待ち画面へ遷移する。
4. 招待参加完了時は自動でホームへ遷移する。
5. 招待された側にもプロフィール未設定フローを必須にする。
6. ホームの `交換する` は対象券の交換確認に直接遷移する。
7. お願いカード全体タップで作成者のみ編集ボタンを表示する。
8. ちゃりん演出後の戻り先はホームにする。
9. ごほうび券カードの残高不足時は `交換する` を非表示にする。
10. Priority 1でスタンプ選択UIを同一画面内の小さなポップオーバーとして表現する。
11. 招待待ち中に招待が期限切れになった場合、同じ画面で新しい招待を再発行する。
12. ちゃりん演出後にホームへ戻っても、10秒取り消しを残り時間だけ継続する。
13. ホームで目標未設定の場合、`ごほうび券を選ぶ` CTAを表示する。
